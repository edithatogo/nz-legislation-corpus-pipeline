# Markdown Style Guide

This document outlines Markdown formatting standards for documentation, READMEs,
and content files in this project.

## File naming

- Use lowercase letters and hyphens for document names.
- Use descriptive names.
- Use `README.md` for repository readmes.

## Headings

- Use ATX style headings.
- Keep heading capitalization consistent and mostly sentence case.
- Include blank lines before and after headings.

## Text formatting

- Use `**bold**` for emphasis.
- Use `*italic*` for emphasis.
- Use backticks for code, file names, and technical terms.

## Lists

- Use `-` for unordered lists.
- Use `1.` for ordered lists.
- Keep nesting shallow and readable.

## Code blocks

- Always specify the language for fenced code blocks.
- Keep code blocks concise and accurate.

## Links

- Use descriptive link text.
- Prefer relative links for repository-local references.

## Tables

- Use pipe tables for simple tabular data.
- Keep columns aligned for readability.

## Accessibility

- Provide alt text for images.
- Use descriptive link text.
- Keep heading levels in order.
