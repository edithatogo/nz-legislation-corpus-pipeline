# Full reconciliation runbook

## Cadence

Run a full reconciliation monthly after the first successful full bootstrap. If the corpus remains small and daily syncs are consistently clean, the maintainer may move the full reconciliation to quarterly, but only after recording that decision in this file or a dated maintenance note.

## Inputs

- `seeds/work_ids.txt`, with provenance comments or adjacent provenance notes.
- Latest Hugging Face `data/manifests/coverage_report.json`.
- Latest Hugging Face `data/manifests/latest_changes.json`.
- Any official NZ Legislation inventory, count, or API documentation update available at the reconciliation date.

## Procedure

The monthly workflow `.github/workflows/monthly_full_reconciliation.yml` automates this procedure. Use manual dispatch for the first run and for any run that may change the seed or upload to Hugging Face.

See `docs/full_corpus_operations.md` for the full-corpus workflow sequence,
host boundaries, and recommended review artifacts.

1. Restore the current Hugging Face dataset state into a clean `data/` directory.
2. Compare `seeds/work_ids.txt` against search discovery output and any official inventory/count source.
3. Add newly verified work IDs to `seeds/work_ids.txt` with provenance.
4. Run the full sync in staged batches when the seed file changes materially.
5. Run `uv run nzlc validate`, `uv run nzlc manifest`, and `uv run nzlc coverage-report`.
6. Review counts by legislation type, status, and year.
7. Review missing text, missing XML URLs, failed versions, and ephemeral identifiers.
8. Record coverage deltas in `latest_changes.json` when generated by the pipeline, or in a dated maintenance note when the explanation requires human context.

## Stop conditions

Do not claim full corpus completeness unless the reconciliation source proves the intended corpus boundary. If the reconciliation source is incomplete, record the gap and keep public wording limited to the verified scope.
